# 👑 黃金法則 (Golden Rules)

1. **有疑問就問！** 開發過程中只要有不確定的地方，絕不瞎猜，立刻發問。
2. **開發和執行遇到問題就是：「截圖、發問、用小畫家開啟」**。遇到未知的錯誤、找不到的畫面，一律先截取最後的現場畫面，用小畫家自動打開給使用者看，然後發問確認，絕對不自行亂點或重啟。
3. **檔案修改邊界**：所有更動的檔案都只能在 `ads/` 目錄內。其他專案資料（如 `src/` 或 `experiments/`）僅供讀取參考，**絕對不進行任何修改**。

# 廣告掃蕩核心策略

根據與使用者的確認，目前的「完整廣告巡邏路線」如下設計：

### 1. 導航階段 (Navigation)
- **進入路徑**：`主畫面` → 點擊 `王國事件` → 點擊 `異界奇聞`。
- **處理方式**：需要從之前的全螢幕截圖中，精準裁切這兩個標籤，放入 `assets/entry/`。

### 2. 掃蕩階段 (Sweep)
- **辨識目標**：進入「異界奇聞」後，畫面上會有多個標有綠色「免費」的 Play 按鈕。
- **點擊順序**：順序不重要，採用 **由左至右、由上至下** 的邏輯，只要有找到，就點擊進入廣告。

### 3. 廣告處理階段 (Watch & Close)
- 點擊「免費」後，進入廣告。
- 前 25-30 秒為**盲等期** (Initial Wait)，不頻繁截圖浪費資源。
- 盲等期結束後，每 1 秒截圖一次，掃描畫面的四個角落。
- 目標是 `assets/ad_close/` 中的 `X` 或 `Skip` 或 `>>` 等按鈕。
- 點擊後，確認畫面是否回到「異界奇聞」。若成功，則重新回到第 2 步，繼續找下一個免費按鈕，直到畫面上的免費按鈕全數消失。

### 4. 異常與防護機制 (Fail-Safe)
如果遇到不可預期的狀況（例如遊戲閃退、不知名的蓋版彈窗、或者找了 60 秒都找不到關閉按鈕）：
1. 絕不盲目亂按 `Back` 或點擊畫面。
2. 將最後畫面存下並打上 FAILED 標記。
3. **自動呼叫小畫家 (`mspaint`)** 打開該截圖。
4. 終止程式碼，留下現場由使用者判定處理。

# 最新實裝成果與視覺辨識突破 (2026-06-05)

## 狀態機流程 (State Machine) 升級
`cli.py` 透過 `AdRunner` 實作了無限循環狀態機，並**移除了全域 120 秒超時**，允許無上限收割廣告：
- `NAV_TO_HUB`: 透過遞迴辨識 (`nav_kingdom.png` -> `nav_otherworld.png` -> 綠色免費按鈕)，不論身處哪一層，都能自動導航到最深處的「異界奇聞」廣告大廳。
- `SWEEP_ADS`: 使用 **HSV 嚴格過濾** 掃描畫面上可用的綠色免費按鈕。
- `TAP_FREE_AD`: 點擊綠色按鈕進入廣告。
- `FIND_CLOSE`: 廣告播放期間盲等 30 秒，接著每 1.5 秒掃描一次畫面尋找 `assets/ad_close/` 目錄下的已知關閉按鈕。若單一廣告超時 (120s)，則觸發黃金法則。
- `VERIFY_RETURN`: 驗證是否成功退出廣告回到大廳。

## 遭遇到的陷阱與突破 (Troubleshooting)
1. **OpenCV 匹配陷阱 (TM_CCOEFF_NORMED)**：
   - **問題**：`cv2.matchTemplate` 對於形狀敏銳，但對顏色不敏感。鑽石卡的灰色「售完」按鈕，竟然和綠色的「免費」按鈕有高達 0.95 的相似度！導致腳本狂點已售完的按鈕。
   - **解法**：放棄單純的 Template Matching，改用 **HSV 色彩空間 (`cv2.inRange`)** 來嚴格過濾「純綠色」。只抓取面積大於 500 且在畫面下半部的綠色輪廓，完美避開灰色按鈕！
2. **「獲得道具」彈窗陷阱 (Dimmed Background)**：
   - **問題**：廣告看完後，遊戲會彈出「獲得道具」視窗，同時會將**背景變暗**。導致綠色按鈕的亮度下降，HSV 遮罩抓不到按鈕，程式誤以為今日任務已完成。
   - **解法**：在 `SWEEP_ADS` 找不到按鈕時，主動點擊畫面左下角安全區域 `(50, 500)` 一次，並等待 1.5 秒，強制關閉任何可能出現的延遲彈窗，然後再次進行 HSV 掃描。
