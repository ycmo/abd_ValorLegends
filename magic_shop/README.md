# 魔法商店全自動掃蕩腳本 (Magic Shop Automation)

這是一個專為《Valor Legends》魔法商店打造的「全自動掃蕩與刷新」模組。它可以精準地辨識商店中的指定高價值商品，自動完成購買（包含多次購買），並在掃蕩完畢後聰明地判斷是否需要花費紅鑽進行商店刷新。

## 🎯 核心目標與特徵
本腳本具備極高的準確度與防呆機制，主要解決了以下痛點：
- **防誤買**：字體太小、太擠導致的 OCR 誤判。
- **防重複點擊**：點擊到已經「售罄 (變灰)」的商品。
- **動態次數購買**：某些商品（如碎片）一包有三個，能自動原地連續購買直到完全售罄。
- **安全財控**：嚴格控管金幣與紅鑽的消耗比例，不會買到破產或花到 200 鑽的漲價刷新。

## 🛡️ 獨家技術：黃金交叉雙重確認 (Golden Cross Validation)
為了達到 100% 的準確率，我們採用了「OCR 大範圍偵測 + 圖片特徵比對」的雙重確認機制：
1. **第一階段 (大範圍掃描)**：利用 `EasyOCR` (放大倍率 `mag_ratio=3.0` + 嚴格白名單) 快速抓出畫面上所有潛在商品的價格標籤。只要信心度達到 `0.35`，即使少看了一個 `k`，也會被放行。
2. **第二階段 (精準圖片把關)**：當 OCR 鎖定一個座標後，程式會拿出該商品的**標準彩色原圖**（例如 `金牌5000k.png`），在那個座標進行 `cv2.matchTemplate` 圖片比對。
   - 如果相似度足夠且**亮度達標** 👉 確認是正確且還沒賣掉的商品，點擊購買！
   - 如果比對失敗或是**按鈕變暗(灰掉)** 👉 代表 OCR 看錯了，或是該商品已售罄，直接略過，絕不誤買！

## 📋 購買目標清單
程式目前會自動鎖定並購買以下商品（包含但不限於）：
- **紫珠**: `960k`
- **競技場券**: `480k`
- **金牌**: `5000k` (對應 OCR 可能遺漏成 `500k`、`5000`、`500`)
- **英雄碎片**: `1800k` (對應 OCR 可能遺漏成 `1800`)

## ⚙️ 執行流程 (Workflow)
1. **單頁掃描與連買**：掃描目前畫面，找出所有目標商品。點擊商品、按下「購買」、關閉「獲得道具」視窗。
2. **原地榨乾機制**：買完一次後，原地再次呼叫圖片比對。如果按鈕還亮著（例如碎片可以買 3 次），就會立刻再買一次，直到按鈕確定變灰為止。
3. **極限滑動**：將畫面上推 `420 像素`（近乎一整頁的高度），只需兩次掃描即可完整讀完一整頁商店的 6 排商品。
4. **安全刷新判斷**：滑到底部後，讀取右上角的「金幣餘額」與「刷新紅鑽按鈕」。
   - **條件**：金幣必須 `>= 12000k`，且右上角刷新按鈕的標籤必須明確寫著 `100`。
   - **動作**：條件符合則自動點擊刷新，並自動尋找畫面上的「是」進行二次確認，隨後進入下一輪。條件不符則安全收工。

## 🚀 如何使用 (Usage)

這個腳本已經封裝為獨立的 CLI 工具，可以在命令提示字元 (Terminal) 中直接執行：

### 1. 全自動掛機模式 (預設)
```powershell
python magic_shop/cli.py
```
> 最推薦的模式。程式會自動買光目標商品，並且自動根據金幣與紅鑽狀況判斷是否刷新。直到金幣小於 12000k 或刷新費漲價為止，才會自動停止。

### 2. 半自動/手動確認模式
```powershell
python magic_shop/cli.py --ask-refresh
```
> 如果你想親自監控它的花費，加上 `--ask-refresh`。每次掃完一頁後，它會告訴你系統判定（建議刷新或不建議），並暫停詢問你：`👉 請問要刷新嗎? (y/n)`。此時只要按下 Enter 就能採用預設建議，或是手動輸入 `y` 進行強制刷新。

### 3. OCR 視覺測試模式 (Dry Run)
```powershell
python magic_shop/cli.py --dry-run
```
> 測試與除錯專用。只會進行 OCR 掃描與圖片比對，**完全不會執行任何點擊或滑動**。它會將 OCR 看到的框線與信心度畫成紅色框框，儲存到 `magic_shop/debug_output/` 資料夾中供你檢閱。

### 4. 動作除錯模式
```powershell
python magic_shop/cli.py --debug
```
> 在背景記錄程式「點了哪裡、滑了哪裡」，會將操作截圖存在 `captures/action_debug/`。
