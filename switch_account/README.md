# 自動切換帳號腳本 (ADB + OpenCV)

這是一個專為手遊掛機設計的全自動帳號切換與登入工具。透過 ADB 指令直接控制 Android 模擬器，並結合 OpenCV 影像辨識技術來突破靜態座標的限制，確保腳本在不同網路速度、伺服器載入時間下的極致穩定度。

---

## 📂 核心架構與檔案

- **`loop_all_accounts.py`**：腳本的**排程中樞**。負責管理所有帳號的循環順序、處理帳號切換間的 60 秒掛機休息時間，並攔截所有異常當機。
- **`switch_account.py`**：腳本的**大腦**。包含所有的影像辨識 (`wait_and_tap`)、畫面流轉判斷，以及不同帳號類型 (Google / 信箱) 的登入邏輯。
- **`accounts.json`**：帳號設定檔。存放各個帳號的類型、信箱、密碼與指定伺服器 (範例可參考 `accounts_template.json`)。
- **`templates/`**：存放所有 UI 按鈕截圖的資料夾，腳本會透過比對這些圖片來判斷目前畫面狀態。

---

## 🚀 核心機制與特色

### 1. 智慧狀態機 (`wait_and_tap`)
這不是單純的「死等 n 秒然後點擊」，而是一個會自我驗證的狀態機：
- 找不到目標時，每隔 2 秒重試一次，直到達到設定的 `timeout`。
- 點擊目標後，會持續掃描畫面，直到確認**按鈕真的因為轉場而消失**，才會放行進入下一步，完美解決網路延遲導致的連點鬼打牆問題。

### 2. 嚴格除錯與防呆機制 (Crash & Inspect)
**「只要遇到看不懂的畫面，就讓程式終止！」**
當腳本發生以下狀況：
1. 尋找某個按鈕超過 `timeout` 秒依然沒出現。
2. 連續點了 3 次同一個按鈕，畫面卻完全不跳轉。
3. 進入遊戲時跑完 10 輪 (約 100 秒) 都沒看到寶箱。
4. 伺服器選擇畫面左右兩邊長得太像，無法分辨。

程式會**立刻拍下當前畫面**、在點擊位置畫上紅框/綠框 (ROI)，存成 `error_debug_*.png` 或 `stuck_debug_*.png`，**並自動彈出「小畫家」給你看**。接著直接拋出 `RuntimeError` 強制中斷流程，絕對不會讓腳本盲點搞砸後續的帳號狀態。

### 3. 強制通過模式 (`force_fallback=True`)
雖然預設機制是嚴格防呆，但 `wait_and_tap` 內建了 `force_fallback` 參數。在未來的開發中，如果你遇到某些「即使沒看到按鈕，也必須強硬點下去」的關卡，加上 `force_fallback=True`，它就會忽略超時報錯，直接盲點你設定的備用座標並強行進入下一步。

### 4. 10 輪動態進入遊戲偵測
登入後的轉場時間往往最不穩定。腳本結尾設計了 10 輪 (每輪 10 秒) 的動態偵測，期間只要捕捉到「進入遊戲按鈕 (006)」或「掛機寶箱 (009)」任一特徵，就能順利收尾。並且針對半透明的進入按鈕設定了底部 `ROI` 限制範圍與較低的 `threshold=0.35`，排除背景動畫干擾。

## 🎮 使用方式

### 基礎設定
1. 確保已開啟 Android 模擬器 (並打開了想掛機的遊戲停留在有頭像的主畫面)。
2. 複製 `accounts_template.json` 並改名為 `accounts.json`，填妥所有的帳號密碼。

### 啟動全部帳號循環
如果你想從頭開始，一次跑遍 `accounts.json` 裡面的所有帳號，請直接在終端機輸入：
```bash
python loop_all_accounts.py
```

### 指定起始帳號
如果腳本中途因為某個帳號當機被中斷了，你想跳過前面的帳號，直接從特定帳號的**下一個**開始接續循環，可以使用 `--start` 參數：
```bash
python loop_all_accounts.py --start tiger
```
*(上述指令會將 `tiger` 設為目前帳號，並立刻從它後面的帳號開始接力執行)*

## 🛠️ 開發與維護指南

### 1. 修改或新增模板圖片 (`templates/`)
- 如果某個按鈕換了樣式，請在遊戲中截圖，盡量只裁切**特徵最明顯**的部分 (例如文字或獨特的圖示邊框)。
- 盡量避免裁切到會動的背景。若無法避免 (例如半透明按鈕)，請在呼叫 `wait_and_tap` 時調降 `threshold` (例如 `0.35`)。

### 2. 備用座標 (`fallback_coord`) 與 ROI
在 `wait_and_tap` 中，`fallback_coord` 不只用來盲點，它還決定了**尋找這張圖片的綠色偵測框 (ROI) 要畫在哪裡**。
- 腳本預設會在 `fallback_coord` 座標周圍切出一個矩形範圍來尋找圖片，大幅節省效能並避免誤判。
- 如果你新增了模板，請務必在 `COORDS` 字典中補上它在螢幕上的 (x, y) 中心座標。
