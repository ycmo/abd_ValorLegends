## 2. 成功與失敗的座標整理

### 成功操作的座標
* **「前往」按鈕掃描**：`x` 座標固定為 `1399` 或 `1400`（按鈕中心點）。
* **「否」按鈕 (關閉退出遊戲彈窗)**：`(385, 745)`。當在遊戲基礎分頁按 Android 返回鍵時，會彈出退出提示，點擊此處可安全關閉。
* **底部主選單「掛機」分頁**：`(800, 840)`。這是返回遊戲大廳/主畫面的核心錨點。
* **主畫面「任務」入口按鈕**：`(1555, 100)`。點擊可直接打開「每日任務」頁面。
* **左上角金黃色返回箭頭**：`(50, 60)`。適用於懸賞委託、公會副本、競技場等多數獨立功能頁面。

### 失敗/未預期的操作座標
* **按鈕縱向過濾 (y >= 400)**：原先為避免誤觸頂部元素而設定，但在列表滾動後，某些任務按鈕滾動到 `y=300` 左右，導致被過濾掉，造成點擊位置與列表項錯位。
* **Android Back Keyevent (keyevent 4)**：在某些特定的深層彈窗或據點頁面（如公會副本的「27號精英據點」）中無效，或者無法在第一時間關閉多層彈窗。

## 3. 返回每日任務頁面的策略

目前測試出最可靠的「返回每日任務頁」序列如下：

1. **發送 Android Back 鍵 keyevent 4**。
2. **點擊 否 (385, 745)** 以關閉退出遊戲提示。
3. **點擊 底部掛機頁籤 (800, 840)**。
4. **點擊 右上任務按鈕 (1555, 100)**。

### 關鍵 UI 載入與等待時間 (`time.sleep`)
* **頁面跳轉等待**：遊戲在進入「秘境副本」、「召喚法陣」等需要加載 3D 資源的場景時，會有短暫黑屏。原先設定的 `3.0` 秒過短，會截圖到加載中的黑屏。已修正為 **`4.5` 秒** 以確保 UI 渲染完畢。
* **返回重定位等待**：每次點擊「掛機」或「任務」按鈕後，應至少等待 `2.0` - `3.0` 秒以防畫面仍在轉場。

## 4. 目前自動化中已解決與未解決的問題

### 已解決的問題
1. **導航中斷修復**：增加了「發送返回鍵 -> 點擊否關閉退出提示 -> 點擊掛機頁籤 -> 點擊任務」的完整導航恢復鏈，即使誤入「英雄列表」等無法直接用返回鍵退出的地方，也能順利召回。
2. **列表錯位問題**：將 `y >= 400` 的過濾條件改為 `150 <= y <= 880` 的範圍限制，保證滾動後處於畫面上方的「前往」按鈕依然能被正確點擊，消成了點擊到背景或進度條的錯誤。

### 未解決/不可靠的問題
1. **任務去重誤判（已在程式碼中修改，待驗證）**：
   * *原機制*：去重時裁剪的寬度為 `200:1000`（寬 800 像素）。這包含了大量相同的深色條狀背景，導致相似度極高（`task_001` 與 `task_002` 的模板匹配相似度高達 `0.9098`）。當設定閾值為 `0.92` 時，極易因些微的對齊位移或渲染抖動而誤判為重複任務跳過（這就是「無盡試煉」被跳過的原因）。
   * *修復方案*：已將裁剪範圍縮小為 `220:650`（寬 430 像素），專注於左側純文字區域，降低背景佔比。
2. **多層/引導彈窗阻擋**：
   * 當首次點擊進入「公會副本」時，會出現一個介紹玩法的半透明全螢幕彈窗。普通的「返回」序列在此處無法順利退出，導致最終停留在「27號精英據點」中，中斷了後續探測。
   * *解決方案*：需要在返回邏輯中，優先檢測並點擊左上角的返回箭頭 `(50, 60)`，它比 `keyevent 4` 更能強制退出這些深層頁面。
