# Valor Legends 人工介入協議文件 (Human Review Protocol)

為確保自動化指令不消耗遊戲資源（如鑽石、召喚券、金幣）、不進行非預期的戰鬥、升級或抽卡，本專案在 `src/actions.py` 中引入了「保守人工介入機制」。

---

## 1. 人工介入觸發情境 (Triggers for STOPPED_FOR_HUMAN_REVIEW)

當執行 `probe-daily-tasks` 探測任務時，系統會在以下 5 種情境下立即拋出 `HumanReviewRequired` 異常，完全終止一切點擊與資源消耗行為，並將現場狀態交給人工處理：

1. **初始啟動狀態錯誤**：
   * *觸發條件*：當執行探測指令時，模擬器目前不在每日任務頁面（檢測信賴度 `< 0.85`）。
   * *說明*：防止在大廳、副本中誤點擊，必須先人工將畫面移回每日任務頁面再啟動。

2. **每日任務頁面檢測處於「灰色模糊地帶」**：
   * *觸發條件*：`daily_tasks_title.png` 匹配信賴度 `confidence` 介於 `0.6` 與 `0.85` 之間。
   * *說明*：此區間說明畫面被其他透明彈窗、引導特效遮擋，或介面有所改變，不應繼續點擊以免錯位。

3. **首頁「前往」按鈕掃描為零**：
   * *觸發條件*：在每日任務第一頁進行按鈕模板匹配時，未找到任何一個 `go_button.png`（偵測按鈕數為 `0`）。
   * *說明*：若首頁沒有任何任務按鈕，可能是數據加載延遲或整個排版錯亂，此時不應盲目向下滾動。

4. **任務去重相似度處於「灰色模糊地帶」**：
   * *觸發條件*：裁剪出的任務描述與已點擊任務庫比對時，最大相似度 `max_val` 介於 `0.85` 與 `0.92` 之間。
   * *說明*：相似度過高可能意味著背景一致但字元有部分重疊，在此信賴度下無法判斷是否為重複任務，必須人工介入防止漏測（先前無盡試煉被誤跳過即屬此類）。

5. **返回每日任務頁面失敗**：
   * *觸發條件*：點擊「前往」進入目標任務頁後，執行完所有復原手段（Back 鍵 -> 否 -> 掛機 -> 任務）後，依然無法在任務頁檢測到 `daily_tasks_title.png`（信賴度 `< 0.85`）。
   * *說明*：表明自動化導航卡死在深層子頁面或遇到新手引導彈窗（如公會副本），必須停止所有操作以防亂點消耗資源。

---

## 2. 人工介入輸出格式 (Console Output Spec)

觸發人工介入時，控制台會印出醒目的標示，並拋出 `HumanReviewRequired` 異常中斷程式：

```
!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
STOPPED_FOR_HUMAN_REVIEW
Reason: [具體原因說明，例如：Failed to return to Daily Tasks page...]
Current Screenshot Path: [E:\antigravity\adb_vl\screenshots\manual_review_YYYYMMDD_HHMMSS.png]
Current Attempted Task / Action: [當前正在嘗試的任務索引或動作名稱]
Attempted Return Strategies: [已經嘗試過的返回路徑列表]
Please review the screen and confirm the next steps manually.
!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
```

此時，系統會**立刻保存現場畫面**至 `screenshots/manual_review_時間戳.png` 供開發者查看。

---

## 3. 人工介入後的除錯與重啟流程

1. **查閱現場截圖**：到 `screenshots/` 下找到最新的 `manual_review_*.png` 截圖，確認模擬器此時卡在何種畫面。
2. **人工手動排除**：
   * 若卡在玩法引導彈窗，手動點擊關閉。
   * 若卡在戰役、據點畫面，手動退回每日任務頁面。
   * 若遇到資源消耗提示，手動點擊「取消」或「否」。
3. **重新執行測試**：確保模擬器回到每日任務頁後，在終端中重新運行 Python 測試。
