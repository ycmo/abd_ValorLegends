# Valor Legends 影像辨識稽核報告 (Vision Audit)

本報告對目前專案自動化程式中的所有畫面判斷、影像辨識、固定座標與安全保護機制進行全面稽核，藉此釐清哪些邏輯具備真正的「電腦視覺能力」，哪些屬於「推定與硬編碼座標」，並給出後續優化建議。

---

## 1. 影像辨識與流程推定的劃分

### A. 真正依賴影像辨識（影像特徵）的判斷
目前專案中，僅有以下判斷是**真正讀取畫面並透過 OpenCV 進行特徵比對**：
1. **每日任務頁面驗證** (`daily_tasks` 頁面判斷)：
   * 比對 `assets/daily_tasks_title.png` 標題特徵。
   * 信心值閾值 $\ge 0.85$ 視為處於每日任務頁。
2. **每日任務「前往」按鈕匹配**：
   * 在掃描任務頁面時，利用 `assets/go_button.png` 在指定的任務滾動區域內比對並標記出所有可用按鈕的像素中心。
3. **任務大廳入口識別**：
   * 比對 `assets/task_button.png` 定位大廳右上角的「任務」圖標。
4. **任務標籤去重（動態模板比對）**：
   * 在點擊「前往」前，程式會動態裁切任務名稱區域 `(Y: target_y-35~target_y+35, X: 220~650)`，與歷史點擊過的任務圖形進行比對，相似度 $\ge 0.92$ 則判定為重複任務。

### B. 依賴固定座標或流程推定的操作
以下操作目前**完全沒有影像比對**，而是直接向 ADB 發送固定座標，或是藉由「跳轉延時後推定」：
1. **關閉遊戲退出彈窗**：
   * 固定點擊 `(385, 745)`（「否」按鈕的物理中心點），不進行彈窗文字或按鈕特徵辨識。
2. **大廳進入挑戰（戰役與無盡試煉入口）**：
   * 戰役關卡大廳挑戰固定點擊 `(1387, 712)`。
   * 無盡試煉野外城堡入口曾使用固定點擊，目前可透過 `assets/endless_trial_label.png` 做模板匹配，若比對失敗則 fallback 到固定點擊 `(790, 535)`。
3. **返回按鈕區域**：
   * 左上角金黃色返回鍵固定點擊 `(50, 60)`。
4. **大廳分頁切換**：
   * 底部五大分頁完全使用等寬分配的物理中心點進行固定點擊：
     * `主城`：`(157, 840)`
     * `野外`：`(477, 840)`
     * `掛機`：`(797, 840)`
     * `英雄`：`(1116, 840)`
     * `神器`：`(1436, 840)`
5. **戰鬥啟動與流程**：
   * 戰役準備按鈕固定點擊 `(1167, 790)`，無盡試煉挑戰按鈕固定點擊 `(1500, 825)`。
   * 戰鬥結束完全靠**時間等待（硬編碼 sleep 25.0 / 30.0 秒）**，沒有對戰鬥勝利/失敗 LOGO 進行實時圖像監控。

## 5. 最小改進建議

為了提升自動化穩定性，並防範在高風險頁面誤操作消耗貴重資源，建議優先實施以下改進：

### A. 最該補的 3 個 Template 檔案
1. **`assets/victory_defeat_title.png`（戰鬥結束標題）**：
   * **用途**：精確監控戰鬥何時結束，在偵測到勝利或失敗圖案時立即可點擊關閉，徹底消除目前盲目等待 `sleep(25)` 或 `sleep(30)` 的延時浪費。
2. **`assets/endless_trial_title.png`（無盡試煉標題）**：
   * **用途**：在進入無盡試煉頁面時，比對左上角標題，100% 驗證已載入該頁面。
3. **`assets/general_close_x.png`（通用關閉 X 按鈕）**：
   * **用途**：自動化辨識彈窗右上角的關閉按鈕位置，避免因解析度微調或彈窗位移導致固定座標 `(1420, 140)` 點擊失效。

### B. 最需要可靠辨識的 2 個高風險 Node
1. **強化裝備/英雄列表 (`hero_list`)**：
   * **原因**：此頁面具備極高金幣與精華消耗風險，且退出時需要複雜的四步回退程序。必須透過影像辨識 100% 確定當前在此頁面，以實行更嚴格的防誤觸限制。
2. **高級契約召喚 (`summon_gate`)**：
   * **原因**：非常容易因為點擊偏移而觸發「召喚x10」或「召喚x1」，造成召喚券或鑽石的大量損耗。必須有明確的頁面辨識以防止盲目執行點擊。

### C. 暫時可保留當作 Fallback 的固定座標
1. **底部 5 大分頁按鈕**：
   * **原因**：底部選單大廳通常保持常駐，且解析度下位置非常固定，在未匹配到分頁名稱時，直接以固定座標切換十分穩定。
2. **「退出遊戲」彈窗的「否」 `(385, 745)` 座標**：
   * **原因**：此系統級確認框位置固定。
